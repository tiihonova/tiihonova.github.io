porting roadmap

pre-production

  • Analyze the project’s game design,architecture.
  • Define the scope of work for the project.
  • Create high-level project documentation.
  • Create initial roadmap for the whole project.
  • Roadmap and documentation sent to partner.

production

  • Choose the right optimization approach.
  • Complete project documentation (GDD, TDD).
  • Create ui mockups for the target platform.
  • Plan changes required for target platforms.
  • Roadmap, documents updated & sent to partner.

first playable

  • Create a rough initial build of the game.
  • Initial platform related adjustments implemented.
  • Analysis of performance & technical blockers.
  • CI/CD system deployed for automated builds.
  • Build sent to partner for review & feedback.

alpha

  • Game is playable, all core features work.
  • Save/Load system implemented.
  • Major UI/UX changes implemented.
  • Significant performance optimization.
  • QA testing on target devices.
  • Build sent to partner for review & feedback.

beta

  • Final UI/UX & performance optimizations.
  • Platform-related services implemented.
  • Multiple region support implemented.
  • Compliance with all platform requirements.
  • Code freeze & final bug fixing done.
  • Build sent to partner for review & feedback.

submission

  • Game is fully optimized
  • Size optimizations if needed
  • All critical bugs are fixed
  • Build submitted for platform review
  • Compliance-related fixes if needed

release

  • Game approved and released.
  • Bug fixing for any issues.
  • Patches\DLCs issued if needed.

closing kit

  • Source code of the game, engine, and
    tools, all documentation archived and
    sent to partner.
